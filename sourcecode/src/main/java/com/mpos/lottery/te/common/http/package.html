Two concepts must be clarified.
  1. Auto Reversal
  2. Auto Retry Request

Auto Reversal

Not all requests can be reversed automatically, below condition must be satisfied:
  1. Client got read timeout.
  2. The transaction must be reversal-able. That says a ReversalHandler is provided when call HttpClientProtoBuffChannel.send(...).
For example, cashout, activation of voucher will be reversed automatically.

Auto Retry Request

If read timeout, current request will be resend automatically till reach the max allowed retry count. Not all requests can be auto retried, it is determined by PaymentTransactionType.isNeedRetryIfTimeout() . At current implementation, only reversal request will be retried.
