Design Principle

The game-type implementation will be represented by a pakcage name 'com.mpos.lottery.te.gameimpl. GAMETYPE ', it should implement the specification defined by package 'com.mpos.lottery.te.gamespec', it includes:
  1. Game Instance Enquiry - In general a game type specific implementation should inherit from com.mpos.lottery.te.gamespec.game.service.impl.AbstractGameInstanceService.
  2. Ticket Sale - A game type specific implementation should inherit from com.mpos.lottery.te.gamespec.sale.service.AbstractTicketService
  3. Payout - A game type specific implementation should implement com.mpos.lottery.te.gamespec.prize.service.PrizeService

ANY CLASSES OUT OF PACKAGE 'com.mpos.lottery.te.gameimpl' MUST NEVER REFERENCE TO ANY CLASSES RESIDE IN THAT GAME-TYPE IMPLEMENTATION PACAKGE('com.mpos.lottery.te.gameimpl').
