Name Convention

  1. Unit testcase must end with 'UnitTest', for example a class 'DateUtil', its corresponding unit testcase class should be 'DateUtilUnitTest'.
  2. Integration test(test the whole web/service/dao layers, or testcase needs external system, such as database) must end with 'IntegrationTest', and extends from com.mpos.lottery.te.test.integration.BaseTransactionIntegrationTest or com.mpos.lottery.te.test.integration.BaseServletIntegrationTest, for example to test sale function, a 'SaleIntegrationTest' will be designed.
  3. Acceptance test(this kind of tests are designed from client's point of view, the test class will access remote service by socket/http/rmi etc) must end with 'AcceptanceTest', for example a 'SaleAcceptanceTest' aims to test sale function.
  4. Performance test must end with 'PerfTest'.
